# 長い詳細版

🗞️ Kamiyama Tribune について


## ■ 一言でいうと

毎朝5時に自動生成される、自分専用の新聞。
AIが世界中の記事から自分の関心領域に合うものを選び、
英国紙風のレイアウトで届けてくれます。


## ■ なぜ作ったか

経営者として日々の情報収集が必要、でも既存のサービスだと足りない。
似たことを試した方も多いと思いますが、それぞれ問題がありました。

「RSSリーダーでいいじゃん」
→ RSSは記事が並ぶだけ。「今日読むべきもの」は選んでくれない。
   時系列で全部流れてくるので、結局読み切れない。
   Tribuneは「編集された朝刊」として、6面構成にまとめてくれます。

「日経電子版とかでいいじゃん」
→ 一紙だけだと視野が偏ります。Tribuneは100以上のソースから
   「特定の媒体に偏らない」設計で記事を集めてきます。
   既に購読している媒体(日経やForesightなど)は意図的に下位扱いにする
   仕組みも入っているので、情報の重複も避けられます。

「Smart NewsとかGunosyでいいじゃん」
→ ニュースアグリゲータは大衆向けで、煽情的なタイトル記事が混ざります。
   Tribuneは英国紙風(Economist や Financial Times)の格調を目指していて、
   低品質コンテンツは設計段階で排除されます。
   具体的には「成功談」「カリスマ礼賛」「金額誇示」系のキーワードに
   ペナルティをかけるブラックリストが入ってます。

「ChatGPTやPerplexityで聞けばいいじゃん」
→ 対話型AIは「質問に答える」のは得意ですが、「今日読むべきものを毎朝
   届ける」体験ではない。Tribuneはプッシュ型の朝刊体験です。
   起きた瞬間に最新版が用意されている、というのが大事。

「自分でも作れそう」
→ 作れます! 実際この合宿のメンバーなら全員作れる前提で、
   別途「あなたのTribuneを作るには」というドキュメントも用意します。
   Claude Codeに指示する形で、約2週間で構築可能です。

つまり、Tribuneは「編集された朝刊」というメタファー自体に意味があって、
RSSやアグリゲータや対話型AIとは思想が違います。


## ■ 紙面構成(全6面 + 編集後記)

第1面: 国際ニュースのトップ記事 + サイドバー解説
第2面: 自社グループ(こころみ・ヒューマンエナジー・ウェブリポ)の業界ニュース
第3面: 6つの領域(地政学/経済/規制/経営/テック/自然科学)から各1記事
第4面: 今週の概念(例: 「我と汝」by ブーバー) + 学術記事
第5面: セレンディピティ枠(普段読まない領域の発見) + AIかみやまのコラム
第6面: 余暇(読書/音楽/アウトドア/料理)
欄外: Tribune 編集部の編集後記(紙面全体を俯瞰した100字)

紙面トップには、東京と高尾山の天気、日経・NYダウ・S&P 500の株価、為替、
「フジロックまで○○日」のカウントダウンまで載ります。


## ■ 特徴と自慢ポイント

1. 二つの人格が紙面に同居
- 「AIかみやま」: 第5面のコラム執筆者、聞き上手で哲学的な対話相手
- 「Tribune 編集部」: 第1面リード文と編集後記の書き手、英国紙風の観察的な声

意図的に2つのvoiceを使い分けて、紙面に厚みを出してます。
AIかみやまは「個人の声」、編集部は「媒体の声」という二層構造。

(注: AIかみやまはTribuneとは別に、神山のブログや社内向け発信資料を
RAGとして構築した分身AIです。Tribuneでは第5面のコラムを毎朝書いてもらってます。
聞き上手で哲学的な対話を得意とする、自分の分身的な存在です。)

2. セレンディピティの設計
普段読まない領域(カルチャー、人類学、書評、音楽、アウトドア等)から
毎朝1記事ランダム選定。過去30日に表示してないジャンルを優先するので、
「同じような話題ばかり」にならない。「自分の関心領域の外」に毎日
出会える設計です。

3. 特定の媒体に偏らない
100以上のRSSソースから記事を集め、ペナルティシステムで「同じ媒体の
連続採用」を回避。既に購読している媒体(日経、Foresight等)は意図的に
下位扱いにして、情報の重複を避けます。

4. AIが記事の核心を凝縮したリード文を生成
第1面トップは記事の核心を60-100字に圧縮した「リード文」を毎朝生成。
元記事を読む前に「なぜこの記事が今朝の紙面トップなのか」が一瞬で掴めます。

5. 完全クラウドで自走
GitHub Actions が毎朝5時に自動起動 → Tribune を生成 → Vercel にデプロイ。
PCを起動する必要なし、何もしなくても毎朝最新版がブックマーク先で待ってます。

6. スマホでも見られる設計
PCでもスマホでも、ブラウザを開けば最新版が表示されます。
ただしスマホ向けのレイアウト最適化(タップしやすいリンクサイズ、
レクタングル表示など)はこれから実装予定。今はPCで読むのがベストです!
