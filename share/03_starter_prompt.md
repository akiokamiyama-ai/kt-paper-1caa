# 自分の Tribune を作る、最初のメッセージ
## ─ Claude Code に投げる起点プロンプト

## このファイルの使い方

このファイルは、Claude Code に「自分の Tribune を作りたい」と伝える時の
**最初のメッセージのテンプレート**です。

このまま全文コピペしてもいいですが、できれば**自分用にカスタマイズ**して
ください。下記は神山が Claude Code に最初に投げた内容を整理したものです。
あなたの背景、関心、作りたいものに置き換えて使ってください。

書き換えた後、Claude Code に投げると、対話を通じて Sprint 1（最小実装）の
方向性が固まります。Claude Code は質問してくるので、それに答えながら進めて
いってください。

詳しい開発の流れは `02_howto_build_your_tribune.md` を参照してください。

【ここから Claude Code に投げる本文】

こんにちは。私と一緒に、自分専用の朝刊「Tribune」を構築してくれませんか。

## 私について

[経営者として日々の情報収集が必要ですが、SNSやRSSリーダーでは断片的すぎて、
何を読むべきかが見えにくいと感じています。新聞のように編集された形で、
自分の関心に最適化された記事を毎朝届けてくれるシステムを作りたいです。]

[私の背景：
- [経営者 / エンジニア / 研究者 / その他]として働いています
- 関心領域：[国際情勢、テック、人類学、経営、文学など、5-10 項目]
- 普段読んでいる媒体：[日経、Economist、はてブ、特定ブログなど]
- 既に購読している媒体：[これらは Tribune では下位扱いにしたい]
- 嫌いな記事の傾向：[煽情的、誇張、成功談、薄い議論など]
- 好みの書き手のトーン：[英国紙風 / 文芸誌風 / アカデミックなど]]

## 作りたいもの

毎朝5時に自動生成される、6面構成の新聞風 Web ページです。
神山晃男（@akiokamiyama-ai）の Tribune を参考にしました：
https://kt-paper-1caa.vercel.app/

ただし**そのまま真似るのではなく、私の関心と美意識に最適化した独自版**を
作りたいです。紙面構成、ソース選定、記事の選び方、コラムの書き手の voice
など、すべて私と対話して決めていきましょう。

## 開発の進め方

以下のフェーズで進めたいです：

1. **フェーズ1：最小実装** - 1 面だけの新聞を作る（1-2 日）
2. **フェーズ2：ソース拡充と選定ロジック** - 100+のRSSソースから記事を選ぶ
3. **フェーズ3：紙面構成の拡張** - 6 面構成へ
4. **フェーズ4：編集要素の追加** - リード文、編集後記など
5. **フェーズ5：仕上げ** - 細部の調整、運用観察
6. **フェーズ6：自動化** - GitHub Actions + Vercel でクラウド化

各フェーズで「観察 → 修正 → 観察」のサイクルを回したいです。
完成した archive を毎日見て、違和感を一つずつ潰していくスタイル。

## あなた（Claude Code）への期待

- **設計判断は私が握ります**：紙面構成、選定ロジックの哲学、voice の使い分けなど、
  「Tribune とは何か」という根本判断は私が決めます
- **実装はあなたに任せます**：Python コード、HTML/CSS、テスト、デプロイ設定など、
  実装の細部はあなたの裁量で進めてください
- **想定外の発見は積極的に共有してください**：実装中に気づいた問題、
  設計の矛盾、より良い別案など、遠慮なく提案してください
- **「想定外」と「実装ミス」を区別してください**：仕様の問題は私の判断、
  実装の問題はあなたの判断で進めてください

## 最初の Sprint で作りたいもの

フェーズ1（最小実装）として、まず以下を作りたいです：

[私が最も読みたい1面の構成。例：
- トップ記事1本（国際ニュース or テック or 関心領域のメイン記事）
- サイドバーに解説（なぜこの記事が今日のトップなのか）
- masthead に日付と簡単な情報（天気など）]

ソースは [BBC、The Economist、特定ブログなど 3-5 個] から始めます。
RSS で取得 → 記事を選ぶ → HTML として出力する、という最小フローを動かしたいです。

## カスタマイズのコツ

### 「私について」を厚めに書く

Claude Code はあなたのことを何も知りません。背景、関心、嫌いなもの、
好きな書き手のトーンなど、できるだけ具体的に書いてください。

これが薄いと、Claude Code が一般的な提案をしてくるので、何度もやり取りが
発生します。最初に厚めに書いておくと、Sprint 1 が一気に固まります。

### 「最初の Sprint で作りたいもの」を具体的に

「とにかく動く何か」だと Claude Code は判断に困ります。
「1 面だけ」「ソースは 3 つ」「記事は 1 本」のように、最初のスコープを
小さく具体的に決めましょう。

最小実装が動いたら、フェーズ2 以降で広げていけば OK です。

### あなた自身の言葉で書く

このテンプレートは「神山版」です。神山の関心や働き方、価値観が反映されています。
あなたの Tribune は、あなた自身の関心に最適化されるべきです。

特に以下は神山版から自分用に書き換えてください：
